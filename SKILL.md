# App Usage Tracker

每小时自动统计 Windows 电脑上各应用的使用时间和 CPU 占用。

## 工作原理

1. **数据采集**：使用 Python psutil 库获取进程数据（无需管理员权限）
2. **实时统计**：即时获取应用内存、CPU、运行时长
3. **时间段分析**：按小时/时间块分析应用使用分布

## 使用方法

### 1. 查看当前应用使用统计

```bash
python get_usage.py
```

输出：按应用汇总的内存占用、CPU峰值、运行时长

### 2. 查看时间段分布

```bash
python get_usage_by_time.py
```

输出：各时间段的活跃应用百分比分布

### 3. 设置定时采集（可选）

通过 cron 设置每小时自动执行 `scripts/collect_usage.py`

## 脚本说明

### get_usage.py

即时统计当前应用使用情况：
- 按应用汇总（合并同名进程）
- 显示：进程数、总内存、CPU峰值、最长运行时间

### get_usage_by_time.py

时间段分析：
- 按小时显示应用分布
- 当前前台应用检测
- 时间块分布（早上/上午/午间/下午/傍晚/晚上）

### scripts/collect_usage.py

定时采集脚本（需设置 cron）：
- 应用名称
- 运行时长
- CPU 占用
- 内存使用量

### scripts/get_daily_report.py

生成每日报告：
- 各应用总使用时间
- CPU 峰值
- 时间分布
- 忙时/闲时分析

## 数据存储

- 定时采集数据：`data/usage_YYYY-MM-DD.json`
- 每日报告：`data/reports/daily_YYYY-MM-DD.md`

## 示例输出

```
=================================================================
📊 应用使用统计 (按应用汇总)
=================================================================
应用名称                         进程数  总内存(MB)   CPU峰值%   最长运行
-----------------------------------------------------------------
chrome.exe                   66     3247.4      0.0      5h 24m
WeChatAppEx.exe              9      211.9       0.0      5h 32m
DingTalk.exe                 4      137.3       0.0      5h 37m
```

## 配置选项

可在 `config.json` 中设置：
- `interval_minutes`: 采集间隔（默认 60）
- `top_processes`: 保留的进程数量（默认 50）
- `exclude_system`: 是否排除系统进程（默认 true）
